# 常见问题

## 余额充足但提示额度不足？

令牌额度和账户余额是分开的。每个令牌有自己的使用上限，即使账户余额充足，令牌额度用完了也会报错。

**解决方法**：在控制台编辑令牌，增加额度或开启无限额度。

## 提示没有可用渠道？

你请求的模型不在当前令牌的分组内，或该分组的所有渠道都不可用。

**解决方法**：换一个包含该模型的分组创建新令牌，或换一个模型试试。

## 常见报错码

- 401 Unauthorized — Token 无效

**检查**：

Token 是否正确复制（没有多余空格）

Token 是否已被删除或过期

请求头格式是否正确：`Authorization: Bearer sk-xxxxx`

- 402 Payment Required — 余额不足

**解决方法**： 登录控制台充值。

- 404 Not Found — 端点不存在

**检查**：

Base URL 是否正确：`{{API_V1_BASE_URL}}`

::: warning /v1 路径检查
注意别多写或少写 `/v1`。
:::

```bash
# ✅ 正确
base_url = "{{API_V1_BASE_URL}}"

# ❌ 错误：会变成 /v1/v1/chat/completions
base_url = "{{API_V1_BASE_URL}}/"
```

- 429 Too Many Requests — 请求太频繁

解决： 等几秒再试，或降低请求频率。

- 500 / 502 / 503 — 服务端错误

可能是上游模型临时不可用。

**解决**：

1. 等一分钟重试
2. 换个模型试试
3. 如果持续出现，联系客服
