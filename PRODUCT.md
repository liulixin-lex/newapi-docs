## Design Context

### Users
GGUU API 文档面向需要接入 API 中转服务的新手和普通开发者。用户通常在配置账号、令牌、CLI 工具、第三方客户端和计费规则时查阅文档，需要快速找到步骤、复制命令、对照截图完成操作。

### Brand Personality
界面应当清晰、可信、克制。语气以实用说明为主，帮助用户降低配置焦虑，避免营销感和过度装饰。

### Aesthetic Direction
采用轻量、专业、文档优先的视觉方向。内容区保持高可读性，截图、表格和代码块要在桌面与移动端都稳定呈现。深色模式和浅色模式都应保持一致的层级、对比度和可扫描性。

### Design Principles
1. 内容优先：所有样式服务于阅读、复制、对照截图和完成配置。
2. 移动端可靠：长代码、长表格和截图不能撑破页面，只允许在局部容器内滚动。
3. 性能克制：图片默认延迟加载并声明尺寸，减少首屏和长页负担。
4. 新手友好：避免隐藏关键步骤，界面文字和视觉层级要帮助用户判断下一步。
5. 发布稳定：优化应尽量小步、可验证，不引入不必要的运行时复杂度。
